Como usar suas chaves API

Autenticação via Header

Authorization: Bearer med_abc123...

Ou usando o header alternativo:

X-API-Key: med_abc123...

📦 1. Buscar Medicamentos

Busca medicamentos por nome comercial ou princípio ativo com fuzzy matching:

GET https://mediddata.com/medicamentos/buscar?q=dipirona

curl "https://mediddata.com/medicamentos/buscar?q=dipirona" \
  -H "Authorization: Bearer med_abc123..."

Resposta: Lista de medicamentos com nome, fabricante, registro ANVISA, preços (PMC/PF/PMVG) e status.

🔍 2. Detalhes de Medicamento

Recupera informações completas de um medicamento específico:

GET https://mediddata.com/medicamentos/{id}

curl "https://mediddata.com/medicamentos/12345" \
  -H "Authorization: Bearer med_abc123..."

Resposta: Dados detalhados incluindo apresentação, via de administração, classe terapêutica, restrições e preços atualizados.

⚠️ 3. Análise de Risco de Prescrição

Valida prescrição médica e detecta riscos de glosa em 9 dimensões:

POST https://mediddata.com/analise/risco
Content-Type: application/json

{
  "paciente": {
    "idade_anos": 65,
    "sexo": "M",
    "peso_kg": 75.5
  },
  "medicamentos": [
    {
      "id": 12345,
      "quantidade": 2,
      "via_administracao": "ORAL"
    }
  ],
  "diagnostico_cid10": "I10",
  "procedimento_sigtap": "0301060029"
}

curl -X POST "https://mediddata.com/analise/risco" \
  -H "Authorization: Bearer med_abc123..." \
  -H "Content-Type: application/json" \
  -d '{...}'

Dimensões analisadas: Registro ANVISA, compatibilidade com CID-10, compatibilidade com SIGTAP, dose por idade/peso, contraindicações, interações medicamentosas, via de administração, custo-efetividade, disponibilidade no mercado.

Resposta: Score de risco (0-100), alertas por dimensão, recomendações e justificativas técnicas.

📊 4. Consultar Consumo

Verifica quanto você já consumiu do seu plano:

GET https://mediddata.com/usuario/consumo

curl "https://mediddata.com/usuario/consumo" \
  -H "Authorization: Bearer med_abc123..."

Resposta: Consumo diário, semanal, mensal e total + limites do plano.

💡 Exemplos de Integração

Python:

import requests

API_KEY = "med_abc123..."
BASE_URL = "https://mediddata.com"

headers = {"Authorization": f"Bearer {API_KEY}"}

# Buscar medicamento
resp = requests.get(
    f"{BASE_URL}/medicamentos/buscar",
    params={"q": "dipirona"},
    headers=headers
)
medicamentos = resp.json()

# Análise de risco
analise = requests.post(
    f"{BASE_URL}/analise/risco",
    json={
        "paciente": {"idade_anos": 65, "sexo": "M"},
        "medicamentos": [{"id": 12345, "quantidade": 1}],
        "diagnostico_cid10": "I10"
    },
    headers=headers
).json()

JavaScript:

const API_KEY = "med_abc123...";
const BASE_URL = "https://mediddata.com";

// Buscar medicamento
const response = await fetch(
  `${BASE_URL}/medicamentos/buscar?q=dipirona`,
  {
    headers: { "Authorization": `Bearer ${API_KEY}` }
  }
);
const medicamentos = await response.json();

// Análise de risco
const analise = await fetch(`${BASE_URL}/analise/risco`, {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${API_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    paciente: { idade_anos: 65, sexo: "M" },
    medicamentos: [{ id: 12345, quantidade: 1 }],
    diagnostico_cid10: "I10"
  })
}).then(r => r.json());

PHP:

$apiKey = "med_abc123...";
$baseUrl = "https://mediddata.com";

$headers = [
    "Authorization: Bearer " . $apiKey,
    "Content-Type: application/json"
];

// Buscar medicamento
$ch = curl_init("$baseUrl/medicamentos/buscar?q=dipirona");
curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$medicamentos = json_decode(curl_exec($ch), true);

🔒 Boas práticas de segurança

  • Nunca compartilhe suas chaves de acesso (nem por e-mail, chat ou repositório público)
  • Revogue imediatamente chaves comprometidas e crie novas
  • Use chaves diferentes para cada ambiente (desenvolvimento, homologação, produção)
  • Armazene chaves em variáveis de ambiente, nunca no código-fonte
  • As chaves respeitam os limites do seu plano - monitore seu consumo regularmente
  • Implemente tratamento de erros para códigos 429 (limite excedido) e 401 (chave inválida)
